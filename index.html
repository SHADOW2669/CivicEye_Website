Civic Eye

Introduction

CivicEye is an advanced AI-powered software designed to detect and monitor traffic violations using CCTV cameras. It enables users to identify infractions such as riding without a helmet or triple riding in real-time. The system seamlessly integrates with existing CCTV setups, processes violations locally for enhanced privacy, and securely transmits critical data to a centralized database. Users can access detailed violation logs, reports, and analytics through an interactive dashboard, making traffic monitoring more efficient and data-driven.

More Info

Objectives

AI-Powered Traffic Violation Detection: Leverage the YOLOv8 object detection model to identify traffic violations such as "No Helmet," "Triple Riding," and other infractions with high accuracy.
Automated Evidence Collection & Reporting: Capture violation details including images, timestamps, and violation types. Automatically store data locally and send reports to a centralized database for further analysis.
Centralized Monitoring & Management: Provide a secure web-based dashboard for users to track, review, and manage recorded violations efficiently.
Privacy-Focused & Efficient Processing: Perform detection directly on the client’s machine, ensuring data privacy, reducing dependency on cloud servers, and lowering operational costs.
Scalable & User-Friendly Deployment: Develop an easy-to-install, lightweight, and adaptable solution that can run across multiple devices with minimal configuration.
Real-Time Detection & Alerts: Provide immediate violation detection and alerts, enabling quick response and action.

Key Features

Real-Time Violation Detection: Automatically detect and classify traffic violations using YOLOv8, ensuring instant identification of rule-breakers.
Local Processing: Perform AI-based detection directly on the client’s machine, reducing reliance on cloud services and ensuring privacy.
Automated Evidence Collection: Capture high-resolution images and metadata (timestamps, vehicle type, violation type) for each detected violation.
Data Reporting & Storage: Store violation logs locally and, if enabled, transmit them securely to a centralized database for further analysis.
Web Dashboard Access: Allow users to review violations, search logs, filter data, and generate reports through a modern, responsive web interface.
Customizable Configuration: Easily set up camera feeds (RTSP URLs) and adjust detection sensitivity to match different environments.
Multi-Violation Support: Detect and categorize multiple types of violations simultaneously, enhancing law enforcement capabilities.
Scalable Deployment: Designed to support small-scale local setups as well as city-wide surveillance systems with minimal hardware requirements.

System Requirements

Operating System: Windows 10 or later (Linux support coming soon).
Processor: Intel i3 or higher (Intel i5/i7 recommended for better performance).
RAM: Minimum 4GB (8GB or more recommended for smooth operation).
GPU (Optional): While the software runs efficiently on CPUs, a dedicated NVIDIA GPU accelerates AI processing and improves detection speed.
Storage: At least 20GB of free disk space for software, logs, and data storage.
Network Connectivity: Required for cloud-based reporting and dashboard access but optional for local-only processing.
Software Dependencies: Requires Python 3.8+, OpenCV, Ultralytics YOLOv8, and AI-related libraries. The installer automatically handles setup.
Camera Support: Supports both USB cameras and IP-based CCTV feeds via RTSP.
